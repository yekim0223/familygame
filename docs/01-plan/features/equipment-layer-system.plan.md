## 1. 배경 및 문제 정의

### 현재 상태
- CharacterSprite는 5개 레이어 구조: L1(배경), L2(오버레이), L3(캐릭터), L4(무기이모지), L5(펫)
- 현재 L4는 무기 이모지(🗡️⚡🔫)를 우하단에 표시하는 단순 뱃지
- 캐릭터 SVG(32×32)에 무기·갑옷이 **그림 내부에 내장**됨
  - `child-warrior.svg` → 이미 검 들고 있음
  - `base-dad.svg` → 슈퍼맨 수트 내장
  - `base-mom.svg` → 마녀 의상 내장

### 목표 상태 (v1)
- base 5종 SVG를 **나신(몸+얼굴만)** 버전으로 교체
- CharacterSprite에 장비 슬롯 **위치 상수** 정의 (미래 overlay 준비)
- 기존 직업 149종 SVG는 **그대로 유지** (v2 이후 점진적 전환)

## 5. 아키텍처 설계

### 5.1 레이어 구조 (미래 완성형)

```
┌─────────────────────────────┐
│ [L5] 투구 SVG overlay        │ ← v2
│ [L4] 무기 SVG overlay (우)   │ ← v2
```
